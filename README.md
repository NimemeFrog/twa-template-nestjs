Шаблон кода для Telegram Web App с использованием NestJS, React + Tailwindcss, TypeScript, Prisma + PostgreSQL, Redis и grammY. <br>
Вы можете брать отсюда любой код и адаптировать его под свой проект. Или же можете скачать этот репозиторий и писать проект на основе него.<br>
Этот шаблон - монорепозиторий, по этому следите за директориями. Не выполняйте команды в корне проекта, они не работают и могут привести к ошибкам.

## Установка
1. Склонируйте репозиторий и перейдите в папку с кодом.
2. Установите зависимости с помощью `npm install` в каждой из папок (<b>не в корне</b>). Если у вас нет Node.js, то установите его из официального сайта.
3. Создайте файл `.env` в каждой из папок (можно использовать команду `cp .env.example .env`).
4. Заполните переменные в файле `.env`, кроме тех, что обновляет скрипт `tunnel.js` (если используете его).
5. Укажите в .env вашу БД (DATABASE_URL). Выполните `npx prisma generate` и `npx prisma migrate dev` в папке `server` для создания таблиц в базе данных (если оставляете Prisma + PostgreSQL).
6. Запустите проект с помощью `npm run dev` в папке `client` и `npm run start:dev` в папке `server`.
7. <i>ОПЦИОНАЛЬНО</i>: Можете использовать скрипт `tunnel.js` для автоматического обновления переменных в файле `.env` при запуске туннеля для локального разработки (только для локальной разработки!). Этот скрипт обновляет URL бекенда и фронтенда для взаимодействия бекенда с TWA. Для TWA нужно HTTPS и localhost.run его предоставляет.

## Аутентификация
Для аутентификации используется только accessToken, это не плохой подход, но если вы пишете проект, в котором на первом месте безопасность, то лучше реализовать refreshToken и обновление accessToken с API сервера.

## I18n
В проекте уже настроен i18n и для TWA и для бота. Если вы не хотите использовать i18n, то можете просто писать всё только в ru.json.

## tunnel.js
Этот скрипт обновляет URL бекенда и фронтенда для взаимодействия бекенда с TWA. Находиться в корне проекта.

## dev.md
Документация по работе с проектом при разработке. Находиться в корне проекта.

## Важно
Конфигурация папки client (`vite.config.ts` и `postcss.config.js`) тонко настроена под React и Tailwind в специфике TWA и может сломаться. Меняйте ее аккуратно.
